# Client Delivery Checklist -- The Titan Genome

> **What you are delivering:** The output of the Titan Genome -- the decoded copywriting DNA of 9 direct response legends, applied to your client's content through the Content Multiplier pipeline. Every piece has been shaped by a Mastermind Table of AI agents trained on a century of proven persuasion architecture. Use this checklist before delivering outputs to a client.

---

## Pre-Delivery QA

### Content Quality
- [ ] All 32 files have passed the Approval Agent (check for "APPROVED FOR PRODUCTION" header)
- [ ] Spot-checked at least 5 files manually for brand voice consistency
- [ ] No placeholder text remaining (`[INSERT]`, `[PLACEHOLDER]`, `{{VARIABLE}}`, `TBD`)
- [ ] No AI-isms present ("In conclusion", "It's important to note", "Let's dive in")

### Brand Consistency
- [ ] Brand name spelled consistently across all 32 files
- [ ] Correct audience names used in correct files (no cross-contamination)
- [ ] All offer names and pricing match CLIENT_CONFIG.md
- [ ] All CTAs point to correct URLs from CLIENT_CONFIG.md
- [ ] Tone and voice consistent with CLIENT_CONFIG.md brand voice description

### Terminology Audit
- [ ] Search all files for preferred terms -- used consistently
- [ ] Search all files for avoided terms -- none found
- [ ] Key concept names are identical across all files
- [ ] No contradictions between files (sales page says X, email says Y)

### Cross-Reference Check
- [ ] Email references to lead magnets match actual lead magnet content
- [ ] Newsletter references to course content match course structure
- [ ] Sales page proof points align with case study content
- [ ] Social media posts reference correct offers and links

### Factual Accuracy
- [ ] All claims are verifiable from the source video/transcript
- [ ] No fabricated statistics, timing claims, or results
- [ ] Tool names, method names, and proper nouns are correct
- [ ] Case study details are accurate and have appropriate permissions

### Formatting
- [ ] All files use consistent markdown formatting
- [ ] Headers, bullets, and spacing are clean
- [ ] Tables render correctly
- [ ] File naming follows convention: `CLIENTNAME_PROJECT_N_Description.md`

---

## Delivery Package

### Files to Include
- [ ] All 32 output files in organized folder structure
- [ ] Content Map (for client reference)
- [ ] Quick Reference card (for client team)

### Files to NOT Include
- [ ] Do NOT include CLIENT_CONFIG.md (contains internal notes)
- [ ] Do NOT include Critic reports or revision history
- [ ] Do NOT include the template files or process documentation
- [ ] Do NOT include the swipe file database

### Delivery Format
- [ ] Files organized in labeled folders by project category
- [ ] Shared via Google Drive / Dropbox / client's preferred platform
- [ ] Delivery email sent with overview and navigation guide
- [ ] Walkthrough call scheduled (if included in service tier)

---

## Post-Delivery

- [ ] Client confirmed receipt
- [ ] Any revision requests documented
- [ ] Revision round completed (if applicable)
- [ ] Final sign-off received
- [ ] Project archived
